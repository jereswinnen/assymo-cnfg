# Phase 4.5 — Material Catalog Filtering Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let tenants toggle which materials from the global registry show up in their configurator. Adds a nullable `tenants.enabled_materials text[]` column (null = unrestricted, empty = nothing enabled, populated = allow-list), a pure domain filter that narrows each catalog, a `PATCH /api/admin/tenants/[id]/enabled-materials` endpoint, and an admin UI — both as a section on `/admin/tenants/[id]` and as a standalone `/admin/registry` page.

**Architecture:** Filtering stays pure and lives in `src/domain/materials/filter.ts`; three small helpers (`filterCatalog`, `filterCatalogAllowing`, and `isMaterialEnabled`) handle the null/array/keep-current-selection cases. `TenantContext` grows a new `enabledMaterials: string[] | null` field; consumers read it via a new `useTenantCatalogs()` hook (in `src/lib/`) that returns memoised wall/roof-trim/roof-cover/floor/door catalogs. The admin UI is a single `MaterialsSection` component reused in two mount points. Existing orders' `quoteSnapshot` is untouched — only future pickers are filtered.

**Tech Stack:** Next 16 (App Router), Drizzle (Neon HTTP), Better Auth, Tailwind v4, shadcn (new-york), Vitest (`vite-plus/test`).

**Spec:** `docs/superpowers/specs/2026-04-17-platform-architecture-design.md` — Phase 4.5 (line 233), Modules table row for `tenant` ("material catalog"), Route map entry `/admin/registry` (line 132), Auth guards table ("Manage material catalog | all | own | —").

**Reference (read-only):** The Phase 4 plan at `docs/superpowers/plans/2026-04-21-phase-4-webshop-shell.md` for style and wave granularity. The existing tenant-scoped PATCH endpoints `src/app/api/admin/tenants/[id]/branding/route.ts` and `.../price-book/route.ts` for the endpoint shape. `src/components/admin/BrandingSection.tsx` for the "Section inside tenant detail" component shape.

## Design decisions (locked)

1. **Schema shape** — `enabled_materials text[]` on `tenants` (nullable). Deferred `tenant_materials` join table until a concrete richer-model need surfaces (YAGNI). Per-material pricing overrides are explicitly out of scope.
2. **Null vs empty array** — `null` = unrestricted (all registry materials allowed). `[]` = explicitly nothing enabled. Default for new + existing rows is `null` (nullable column, no backfill).
3. **UI location** — BOTH: `MaterialsSection` is mounted on `/admin/tenants/[id]` next to Branding and PriceBook (super_admin + tenant_admin), AND a standalone `/admin/registry` route exists for tenant_admin as a first-class nav target. Super_admin hitting `/admin/registry` is redirected to `/admin/tenants` because super_admin has no single "own" tenant.
4. **Categorisation** — Group toggles per catalog (Wanden / Dakbedekking / Daktrim / Vloeren / Deuren). A slug appearing in multiple catalogs shows up in multiple groups; toggling either syncs to the single global set (checkboxes reflect that).
5. **Scene compatibility** — The atom registry stays unrestricted. `getAtom()` / `getAtomColor()` pass through disabled slugs (so scenes with already-selected disabled materials still render). Pickers filter catalogs but keep the current selection visible via `filterCatalogAllowing(..., currentValue)` so users can see what's currently picked and can switch to any enabled alternative.
6. **No revalidation of historical orders** — `quoteSnapshot` frozen at submit stays frozen; Phase 4.5 only affects new pickers.
7. **Sentinel materials** — The `geen` floor atom (`isVoid: true`) remains unconditionally available regardless of `enabledMaterials` because it represents "no floor"; hiding it would break the configurator. Implemented in `filterCatalog` as a small allow-list exception.

```ts
import type { BaseCatalogEntry } from './types';

/** Slugs that must always show in pickers regardless of tenant filtering.
 *  `geen` is the "no floor" sentinel — hiding it would break the floor
 *  picker when a building has no floor (isVoid). */
export const ALWAYS_ENABLED_SLUGS = ['geen'] as const satisfies readonly string[];

/** Pure predicate: can this tenant use this material slug?
 *  `enabled === null` means unrestricted — all slugs allowed. */
export function isMaterialEnabled(
  slug: string,
  enabled: readonly string[] | null,
): boolean {
  if (enabled === null) return true;
  if ((ALWAYS_ENABLED_SLUGS as readonly string[]).includes(slug)) return true;
  return enabled.includes(slug);
}

/** Filter a per-category catalog against a tenant's enabled list.
 *  Null input returns the catalog reference verbatim (useful for
 *  memoisation in the browser). Sentinels in ALWAYS_ENABLED_SLUGS
 *  are preserved even when absent from the enabled list. */
export function filterCatalog<T extends BaseCatalogEntry>(
  catalog: readonly T[],
  enabled: readonly string[] | null,
): readonly T[] {
  if (enabled === null) return catalog;
  const allowed = new Set<string>(enabled);
  for (const s of ALWAYS_ENABLED_SLUGS) allowed.add(s);
  return catalog.filter((entry) => allowed.has(entry.atomId));
}

/** Like filterCatalog, but always keeps the currently-selected slug
 *  visible in the picker (even if it has been disabled for the tenant
 *  after the scene was created). Lets existing scenes stay editable
 *  without being forced onto a different material. */
export function filterCatalogAllowing<T extends BaseCatalogEntry>(
  catalog: readonly T[],
  enabled: readonly string[] | null,
  currentAtomId: string | null,
): readonly T[] {
  if (enabled === null) return catalog;
  const allowed = new Set<string>(enabled);
  for (const s of ALWAYS_ENABLED_SLUGS) allowed.add(s);
  if (currentAtomId) allowed.add(currentAtomId);
  return catalog.filter((entry) => allowed.has(entry.atomId));
}
```

```ts
import { MATERIALS_REGISTRY } from '@/domain/materials/atoms';

/** Per-tenant material allow-list. `null` = unrestricted; any array
 *  (including empty) is interpreted literally. */
export type EnabledMaterials = string[] | null;

export interface ValidatedEnabledMaterialsPatch {
  enabledMaterials: EnabledMaterials | undefined;
  errors: string[];
}

function isObject(v: unknown): v is Record<string, unknown> {
  return typeof v === 'object' && v !== null && !Array.isArray(v);
}

/** Validate a PATCH body for the enabled-materials endpoint. Mirrors
 *  the shape of validateBrandingPatch / validatePriceBookPatch:
 *  returns the cleaned value plus a list of field paths that failed.
 *  Empty errors == safe to store. Slugs must exist in the registry
 *  to catch typos early; the tenant can't enable a material that
 *  doesn't exist yet. */
export function validateEnabledMaterialsPatch(
  input: unknown,
): ValidatedEnabledMaterialsPatch {
  if (!isObject(input)) {
    return { enabledMaterials: undefined, errors: ['body'] };
  }

  if (!('enabledMaterials' in input)) {
    return { enabledMaterials: undefined, errors: ['enabledMaterials'] };
  }

  const raw = input.enabledMaterials;

  if (raw === null) {
    return { enabledMaterials: null, errors: [] };
  }

  if (!Array.isArray(raw)) {
    return { enabledMaterials: undefined, errors: ['enabledMaterials'] };
  }

  const errors: string[] = [];
  const seen = new Set<string>();
  const out: string[] = [];
  for (let i = 0; i < raw.length; i += 1) {
    const slug = raw[i];
    if (typeof slug !== 'string' || slug.length === 0) {
      errors.push(`enabledMaterials[${i}]`);
      continue;
    }
    if (!(slug in MATERIALS_REGISTRY)) {
      errors.push(`enabledMaterials[${i}]`);
      continue;
    }
    if (seen.has(slug)) continue;
    seen.add(slug);
    out.push(slug);
  }

  if (errors.length > 0) {
    return { enabledMaterials: undefined, errors };
  }
  return { enabledMaterials: out, errors: [] };
}
```

```ts
export const PATCH = withSession(
  async (session, req, ctx: { params: Promise<{ id: string }> }) => {
    requireBusiness(session, ['super_admin', 'tenant_admin']);
    const { id } = await ctx.params;
    requireTenantScope(session, id);

    let body: unknown;
    try {
      body = await req.json();
    } catch {
      return NextResponse.json({ error: 'invalid_json' }, { status: 400 });
    }

    const { enabledMaterials, errors } = validateEnabledMaterialsPatch(body);
    if (errors.length > 0) {
      return NextResponse.json(
        { error: 'validation_failed', details: errors },
        { status: 422 },
      );
    }

    const [current] = await db
      .select()
      .from(tenants)
      .where(eq(tenants.id, id))
      .limit(1);
    if (!current) {
      return NextResponse.json({ error: 'tenant_not_found' }, { status: 404 });
    }

    const [updated] = await db
      .update(tenants)
      .set({
        enabledMaterials: enabledMaterials ?? null,
        updatedAt: sql`now()`,
      })
      .where(eq(tenants.id, id))
      .returning();

    return NextResponse.json({ tenant: updated });
  },
);
```

Note on `enabledMaterials ?? null`: when the validator returns `null`, `??` preserves it; when it returns `[]`, that's truthy-enough for `??` to keep it; when it returns `undefined` (should be impossible past the `errors.length > 0` guard) we defensively fall back to `null`. TypeScript narrows this correctly because `enabledMaterials` is `string[] | null | undefined` and the `??` collapses `undefined`.

```tsx
  id: 'wall' | 'roofCover' | 'roofTrim' | 'floor' | 'door';
  labelKey: string;
  catalog: readonly BaseCatalogEntry[];

const GROUPS: Group[] = [
  { id: 'wall', labelKey: 'admin.registry.group.wall', catalog: WALL_CATALOG },
  { id: 'roofCover', labelKey: 'admin.registry.group.roofCover', catalog: ROOF_COVERING_CATALOG },

/** Count how many catalogs each slug appears in — used to render a
 *  hint when the admin toggles a slug that affects multiple pickers. */
function buildSlugCatalogCount(): Map<string, number> {
  const m = new Map<string, number>();
  for (const g of GROUPS) {
    for (const entry of g.catalog) {
      m.set(entry.atomId, (m.get(entry.atomId) ?? 0) + 1);
    }
  }
  return m;
}

export function MaterialsSection({ tenantId, initialEnabledMaterials }: Props) {
  const [unrestricted, setUnrestricted] = useState(initialEnabledMaterials === null);
  // When `unrestricted`, the set is a ghost copy: we keep whatever the
  // admin had selected so flipping the switch back restores it. When
  // `null` initially, we seed with "everything currently in catalogs".
  const [enabledSet, setEnabledSet] = useState<Set<string>>(() => {
    if (initialEnabledMaterials === null) {
      return new Set(Object.keys(MATERIALS_REGISTRY));
    }
    return new Set(initialEnabledMaterials);
  });
  const [busy, setBusy] = useState(false);
  const [msg, setMsg] = useState<string | null>(null);

  const slugCatalogCount = useMemo(buildSlugCatalogCount, []);
  const sentinelSlugs = useMemo(
    () => new Set<string>(ALWAYS_ENABLED_SLUGS),
    [],

    setEnabledSet((prev) => {
      const next = new Set(prev);
      if (checked) next.add(slug);
      else next.delete(slug);
      return next;

      enabledMaterials: unrestricted ? null : Array.from(enabledSet).sort(),

              <ul className="grid grid-cols-1 sm:grid-cols-2 gap-2">

                  const checked = sentinelSlugs.has(slug) || enabledSet.has(slug);
                  const shared = (slugCatalogCount.get(slug) ?? 0) > 1;
                  const locked = sentinelSlugs.has(slug);
                  return (

                      key={`${group.id}:${slug}`}
                      className="flex items-start gap-2 rounded-md border border-border/60 px-3 py-2"

                      />
                      <div className="flex-1 min-w-0">
                        <Label

                          className="cursor-pointer text-sm"
                        >
                          <span

                            style={{ backgroundColor: atom?.color ?? '#ccc' }}
                            aria-hidden
```

- **Type consistency:** `EnabledMaterials` is `string[] | null`. `TenantContext.enabledMaterials` uses the same shape directly rather than importing the alias (keeps `types.ts` free of the `enabledMaterials.ts` circular dep risk — `enabledMaterials.ts` imports from `@/domain/materials/atoms`, not from `./types`). The API route reads `tenant.enabledMaterials` straight off the `TenantRow` (whose type is inferred from the Drizzle column, `text(...).array()` → `string[] | null`).
- **No placeholders:** every code-bearing step shows complete code. Test step counts are explicit so a subagent can verify.
- **Framework boundaries:** `filter.ts` and `enabledMaterials.ts` have ZERO React/Next/Drizzle imports. `enabledMaterials.ts` imports `MATERIALS_REGISTRY` from another domain module, which is pure — safe. The `useTenantCatalogs` hook is the single bridge from the browser to the pure helper; admin/configurator components depend on the hook, not on `filter.ts` directly.
- **Auth guard reuse:** the new endpoint reuses `requireBusiness(['super_admin','tenant_admin'])` + `requireTenantScope`, identical to branding/price-book. No new guard, no new tests for the guard path (covered at the guard level already).
- **Rules-of-Hooks:** every consumer calls `useTenantCatalogs` UNCONDITIONALLY at the top of its component body, before any early returns. Tasks 12 (FloorConfigSection) and 13 (DoorConfig) and 14 (SurfaceProperties) explicitly call out this ordering.
- **Data-integrity for existing scenes:** `filterCatalogAllowing(catalog, enabled, currentSelection)` guarantees the current pick stays visible. The atom registry is NOT filtered — `getAtom()` / `getAtomColor()` still return textures / colours for disabled slugs, so 3D rendering of legacy scenes is unaffected.
- **Risk surface — the Drizzle migration is the riskiest change:** altering a production table. The migration is additive (nullable column, no default, no backfill), which Postgres applies as an O(1) metadata-only operation on Neon. No downtime concerns. Task 6 Step 2 demands verification that the generator emits exactly one column ALTER and nothing else — if it tries to touch other tables, STOP.
