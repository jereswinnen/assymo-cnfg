Standalone Walls — Entity Design Overview

New building type 'muur' following the pole pattern

How it works

Data Model
type: 'muur'
position: [x, z]
dimensions: {
width: 3m ← long side
depth: 0.15m ← thickness
height: inherited ← follows buildings
}
walls: { 'front': WallConfig } ← one face
orientation: 'horizontal' | 'vertical'
Height Inheritance
1. Buildings define their height (default 3m)
2. Walls & poles follow by default
3. User can override per entity
When building height changes:
Non-overridden walls/poles update automatically

2D Schematic View

Key Behaviors

📐
Axis-aligned
Horizontal or vertical only. Toggle orientation with a button or context menu.
🧲
Smart snapping
Snap to building edges, poles, and other walls. Edge-slide + detent like poles.
🎨
Full wall config
Same materials, doors, windows, and finishes as building walls.
